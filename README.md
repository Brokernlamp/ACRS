# AI Growth Operator

Marketing analytics platform for agencies — predictive insights, budget optimization, AI chatbot, automated reporting.

## Stack
| Layer | Tech |
|---|---|
| Frontend | Next.js 16, TypeScript, Tailwind CSS |
| Backend | FastAPI, Python 3.10+ |
| Database | SQLite (dev) / PostgreSQL (prod) |
| AI | Google Gemini 2.0 Flash Lite |
| RAG | ChromaDB + all-MiniLM-L6-v2 |
| Charts | Plotly → PNG |
| PDFs | ReportLab |

## Local Development

**Requirements:** Python 3.10+, Node.js 18+

```bash
git clone <repo-url>
cd ACRS
chmod +x setup.sh dev.sh
./setup.sh    # one-time setup
./dev.sh      # start both servers
```

Open **http://localhost:3000**

## Production Deployment

### Step 1 — Deploy Backend on Render

1. Push your code to GitHub

2. Go to [render.com](https://render.com) → **New** → **Blueprint**

3. Connect your GitHub repo — Render will detect `render.yaml` automatically

4. It will create:
   - A **Web Service** (`acrs-backend`) running FastAPI
   - A **PostgreSQL database** (`acrs-db`) — free tier

5. After deploy, go to the `acrs-backend` service → **Environment** tab and add:

   | Key | Value |
   |---|---|
   | `GEMINI_API_KEY` | Your Gemini API key |
   | `ALLOWED_ORIGINS` | `https://your-app.vercel.app` (fill in after Step 2) |
   | `SMTP_USER` | Your Gmail (optional) |
   | `SMTP_PASSWORD` | Gmail App Password (optional) |

6. Note your backend URL: `https://acrs-backend.onrender.com`

> **Note:** The free Render plan sleeps after 15 minutes of inactivity. First request after sleep takes ~30s. Upgrade to Starter ($7/mo) to keep it always-on.

---

### Step 2 — Deploy Frontend on Vercel

1. Go to [vercel.com](https://vercel.com) → **New Project**

2. Import your GitHub repo

3. Set **Root Directory** to `frontend`

4. Add environment variable:

   | Key | Value |
   |---|---|
   | `NEXT_PUBLIC_API_URL` | `https://acrs-backend.onrender.com` |

5. Click **Deploy**

6. Note your frontend URL: `https://your-app.vercel.app`

---

### Step 3 — Connect Frontend ↔ Backend

1. Go back to Render → `acrs-backend` → **Environment**
2. Update `ALLOWED_ORIGINS` to your Vercel URL:
   ```
   https://your-app.vercel.app
   ```
3. Render will auto-redeploy

## Environment Variables

### Backend (`backend/.env` locally, Render dashboard in production)

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | Auto-set by Render. Locally: `sqlite:///./acrs.db` |
| `SECRET_KEY` | Yes | Random 64-char string. Auto-generated by Render |
| `JWT_SECRET_KEY` | Yes | Random 64-char string. Auto-generated by Render |
| `ALLOWED_ORIGINS` | Yes | Comma-separated frontend URLs |
| `GEMINI_API_KEY` | No | Enables AI chatbot |
| `SMTP_USER` | No | Gmail for email reports |
| `SMTP_PASSWORD` | No | Gmail App Password |

### Frontend (`frontend/.env.local` locally, Vercel dashboard in production)

| Variable | Required | Description |
|---|---|---|
| `NEXT_PUBLIC_API_URL` | Yes | Full URL of your FastAPI backend |

## CSV Format

```csv
date,campaign,impressions,clicks,spend,leads,revenue
2024-01-01,Google Search,50000,600,320.50,45,5400.00
2024-01-01,Meta Retargeting,30000,180,150.00,12,1440.00
```

`revenue` column is optional — enables ROAS calculation.
